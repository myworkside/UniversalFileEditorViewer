# Security Policy

## Supported Versions

The latest version of Universal File Editor & Viewer receives security updates.

## Reporting a Vulnerability

If you discover a security vulnerability, please do not create a public issue.

Instead:

1. Use GitHub Private Vulnerability Reporting.
2. Contact the maintainer privately.
3. Include:

   * Description of the issue
   * Steps to reproduce
   * Android version
   * Device model
   * Screenshots if applicable

## Response Time

Security reports are reviewed as quickly as possible.

## Scope

Examples of security issues:

* Unauthorized file access
* Permission bypass
* FileProvider vulnerabilities
* Path traversal issues
* Data leakage
* Exported component vulnerabilities
* Unsafe URI handling

Thank you for helping improve the security of this project.
